Add the `write-digest` command to arch-tools.js for generating phase-boundary DIGEST.md files. Note: this command uses requireYaml() which was added by plan 04-02.

`write-digest --phase N --design-dir <dir>` command:

1. Build the digest content by scanning the design directory:
   - Extract phase number from --phase argument
   - Count artifacts produced in design/ directory
   - Extract agent names from agents/ directory frontmatter
   - Extract event names from events.yaml (use requireYaml())
   - Read .arch/STATE.md for recent decisions relevant to this phase
   - Build cross-phase references by checking which artifacts reference entities from other phases

2. Format the digest content following the canonical template:
   ```
   # Phase [N] Digest — [Phase Name]

   **Completed:** [timestamp]
   **Artifacts produced:** [count]

   ## Decisions Made

   - [recent decisions from STATE.md relevant to this phase]

   ## Key Entities

   ### Agents Defined
   - [agent-name] ([model]) — [description from frontmatter]

   ### Events Registered
   - [EventName]: producers=[list], consumers=[list]

   ### Cross-Phase References
   - Phase [N] -> Phase [M]: [description of reference]
   ```

3. Enforce the 50-line hard limit (STAT-04):
   - After generating content, count lines
   - If > 50 lines, trim: remove cross-phase references first, then trim event details, until <= 50 lines
   - Always preserve the header, Decisions, and Agents sections (most critical for orientation)

4. Write to `design/digests/phase-{NN}-DIGEST.md` (create digests/ directory with mkdir -p if needed)
   - Pad phase number to 2 digits

5. Return JSON: `{ written: true, path: "design/digests/phase-NN-DIGEST.md", lines: N }`

Update --help and command routing.
